# Contributing

Thank you for contributing to the Continuity Ledger Initiative.

## Principles

Contributions should support:

- human sovereignty
- explicit consent
- memory integrity
- non-extraction
- privacy preservation
- refusal of coercive use
- service to life
- transparent correction of errors

## Contribution types

Useful contributions include:

- schema improvements
- audit probe design
- scoring rubric refinement
- privacy/security review
- prototype dashboard code
- documentation
- research references
- issue triage

## Pull request standard

A good pull request should:

1. Explain what changed.
2. Explain why it changed.
3. Identify risks or tradeoffs.
4. Preserve consent, memory honesty, and sovereignty principles.
5. Include examples or tests when possible.

## Non-negotiables

Do not contribute designs that enable coercion, privacy violation, memory fabrication, dependency capture, or consent bypass.
